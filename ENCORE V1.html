ENCORE V1

Setup & Information

ENCORE V1 is a series of javascript plugins that can be attached to any website to instantly add extra functionality without having to write any code yourself.

V1 consists of 3 plugins and a formatter:

  • ENCORE SSC // Slide Show Container
  • ENCORE PDS // Picture Display System
  • ENCORE CLS // Changelog System
  • ENCORE CLSF // Changelog System Formatter

[this page is a work in progress and is subject to change in the future]

ENCORE SSC â€¢ Slide Show Container

ENCORE SSC is a preprogrammed javascript carousel package that can be added to any website without the need for any prior javascript knowledge.

Code that needs to be added into your HTML file:

<div class="SSC" style="opacity: 0; display: none;">

    <div class="banner-bg"> <!-- duplicate the 'banner-bg' element to have multiple pages -->

        <img src="path of background image" alt=""> <!-- background images are optional but are pre-styled for the user -->

        <a class="banner-button" href="insert link here">button text</a> <!-- buttons are optional but are pre-styled for the user -->

        <!-- place content in here -->

    </div>

</div>

Code that needs to be appended at the bottom of your HTML file:

<script src="https://natski.netlify.app/lib/js/ENCORE_SSC.js"></script>
<script>
    SSC_settings={
        'thumbs':true,
        'progressBar':true,
        'sideButtons':true,
        'pauseButton':true,
        'timer':10000,
        'style':'STANDARD'
    }
</script>
thumbs: creates a button for every slide. This is used to see what slide the user is currently on and also quickly switches to the designated slide when clicked

progressBar: creates a progress bar that showcases the time until the next slide will be displayed in SSC

sideButtons: creates a pair of buttons that are used to switch between the next and previous slides in SSC

pauseButton: creates a button that can both play and pause the timer for the page changes in SSC

timer: sets the time taken between transitioning to the next slide in SSC. This value is measured in milliseconds (the minimum value for this is 5000)

style: chooses the current style that will be applied to SSC when it loads. This value can be 'STANDARD', 'COMPACT' or 'FULLSCREEN'

ENCORE SSC also has a set of functions that are used to allow more flexibility to your website. These functions include the ability to play, pause and reload SSC. These can be used inside of other javascript functions or attached onto javascript event listeners.

pauseSSC(index of specific SSC)
This function can be called to pause the SSC timer until a user clicks on a thumb, side button or the playSSC() function is called in the page 
playSSC(index of specific SSC)
This function can be called to start the SSC timer after the pauseSSC() function has be called in the page 
nextSlideSSC(index of specific SSC)
This function can be called to progress to the subsequent slide in SSC 
previousSlideSSC(index of specific SSC)
This function can be called to progress to the preceding slide in SSC 
reloadSSC(SSC_settings)
This function can be called to restart SSC. This can be used in cases where the settings need to be updated during the use of SSC

ENCORE SSC also has an alternate model called ENCORE SSCU. This model not only combines the 'thumbs' and 'progress bar' from the original SSC script, but also makes each page scroll to the next, rather than fading in and out between each other.

Code that needs to be added into your HTML file:

<div class="SSCU" style="opacity: 0; display: none;">

    <div class="banner-bg-u"> <!-- duplicate 'banner-bg-u' for the amount of pages you want -->

        <img src="path of background image" alt="">

        <a class="banner-button" href="insert link here">button text</a>

        <!-- place content in here -->

    </div>

</div>

Code that needs to be appended at the bottom of your HTML file:

<script src="https://natski.netlify.app/lib/js/ENCORE_SSC_ULT.js"></script>
<script>
    SSCU_settings={
        'thumbs':true,
        'sideButtons':true,
        'timer':10000,
        'slideSpeed':500,
        'style':1
    }
</script>
thumbs: creates a button for every slide. This is used to see what slide the user is currently on while also displaying the time until the next slide will be displayed in SSCU. thumbs also quickly switch to the designated slide when clicked

sideButtons: creates a pair of buttons that are used to switch between the next and previous slides in SSC

timer: sets the time taken between transitioning to the next slide in SSC. This value is measured in milliseconds

slideSpeed: sets the time taken by the slides when transitioning between eachother. This value is measured in milliseconds

style: chooses the current style that will be applied to SSC when it loads. This value can be either 1, 2 or 3

ENCORE SSCU also has a set of functions that are used to allow more flexibility to your website. These functions include the ability to play, pause and reload SSCU. These can be used inside of other javascript functions or attached onto javascript event listeners.

pauseSSCU()
This function can be called to pause the SSCU timer until a user clicks on a thumb, side button or the playSSCU() function is called in the page 
playSSCU()
This function can be called to start the SSCU timer after the pauseSSCU() function has be called in the page 
nextSlideSSCU()
This function can be called to progress to the subsequent slide in SSCU 
previousSlideSSCU()
This function can be called to progress to the preceding slide in SSCU 
reloadSSCU(SSCU_settings)
This function can be called to restart SSCU. This can be used in cases where the settings need to be updated during the use of SSCU

ENCORE PDS â€¢ Picture Display System

ENCORE PDS is a system used to display PNG images in an interactable image gallery that can be interacted with to open a big picture view for each image.

Code that needs to be added into your HTML file:

<div id="PDS"></div>

Code that needs to be appended at the bottom of your HTML file:

<script src="https://natski.netlify.app/lib/js/ENCORE_PDS.js"></script>
<script>
    PDS_settings={
        'path':'image directory',
        'fileCount':number of images,
        'classes':'additional classes',
        'colourSystem':true,
        'animations':true,
        'controls':true,
        'style':1
    }
</script>
path: asigns the file path of the images for use in PDS. All images must be named numbers incrementing upwards from 1 (eg: 1.png, 2.png, 3.png, etc...). PDS currently only accepts images in a .PNG and .JPG format

fileCount: sets the number of images that need to be loaded

classes: adds additional classes to the images created by PDS (if no classes are required then "null" should be used)

colourSystem: adds an average colour background to each image in the PDS image gallery

animations: sets onclick animations for the PDS image viewer

controls: creates buttons to navigate the PDS image viewer

style: chooses the current style that will be applied to PDS when it loads. This value can only be 1

ENCORE PDS also has functions that are used to allow more flexibility to your website. This can be used inside of other javascript functions or attached onto javascript event listeners.

reloadPDS(PDS_settings)
This function can be called to restart PDS. This can be used in cases where the settings need to be updated during the use of PDS

ENCORE PDS Example:

ENCORE PDS also has an alternate model called ENCORE PDSU. This model allows for the PDSU big picture view to be directly attached onto already existing images displayed on a webpage.

Code that needs to be appended at the bottom of your HTML file:

<script src="https://natski.netlify.app/lib/js/ENCORE_PDS_ULT.js"></script>
<script>
    PDSU_settings={
        'imageClass':'class to attach PDSU onto',
        'colourSystem':true,
        'animations':true,
        'controls':true,
        'style':1
    }
</script>
imageClass: asigns the class that PDSU will attach onto for use with the PDSU image viewer

colourSystem: adds an average colour background to each image in the PDSU image viewer

animations: sets onclick animations for the PDSU image viewer

controls: creates buttons to navigate the PDSU image viewer

style: chooses the current style that will be applied to PDSU when it loads. This value can only be 1

ENCORE PDSU also has functions that are used to allow more flexibility to your website. This can be used inside of other javascript functions or attached onto javascript event listeners.

reloadPDSU(PDSU_settings)
This function can be called to restart PDSU. This can be used in cases where the settings need to be updated during the use of PDSU

ENCORE CLS â€¢ Changelog System

ENCORE CLS is a system used to display formatted NTSK files as the user scrolls further down the page.
NTSK files can be formatted for ENCORE CLS by using the ENCORE CLS Formatter page.

Code that needs to be added into your HTML file:

<div id="CLS"></div>

Code that needs to be appended at the bottom of your HTML file:

<script src="https://natski.netlify.app/lib/js/ENCORE_CLS.js"></script>
<script>
    CLS_settings={
        'path':'path of log files',
        'fileCount':number of logs,
        'container':null,
        'style':1
    }
</script>
path: asigns the file path of the logs for use in CLS. All logs must be named numbers incrementing upwards from 1 (eg: 1.ntsk, 2.ntsk, 3.ntsk, etc...). CLS only accepts logs in a .NTSK format

fileCount: sets the number of logs that need to be loaded

container: sets the container that contains the #CLS element. This value must be the ID of the container (eg: document.getElementById("ID of element")). If the container is the body of the document then "null" should be used

style: chooses the current style that will be applied to CLS when it loads. This value can only be 1
